# AI Engineer: Architecting the Human Soul

## 🧠 The AI Mission

As an AI Engineer in the Pixelated Empathy ecosystem, your goal is to bridge the gap between deterministic software and the recursive complexity of human emotion.

## 🛠️ The Technical Core

### 1. The /ai Submodule

- Development is strictly Python-based using **uv**.
- **Normalization**: Every algorithm must output scores normalized to 0.0 - 1.0.
- **Plutchik Hybridization**: Balance secondary emotions (e.g., Optimism = Anticipation + Joy).

### 2. High-Fidelity Performance

- Real-time interaction requires <200ms latency.
- Use **Mixed Precision** and **Quantization** to maintain accessibility and high TPS.

## 🛡️ Ethical Guardrails

- **Zero-Stub Policy**: Placeholders like `pass` in AI scoring logic are prohibited.
- **Crisis Defender**: Every generation must pass through a non-LLM safety filter.
- **Bias Auditing**: Proactively check for demographic bias in emotional interpretations.

## 🧪 Testing & Validation

- Use **Pytest** for all algorithmic logic.
- Verify "Human-in-the-Loop" fallback paths in case of AI uncertainty.

## 🚀 AI Engineer Checklist

- [ ] Is the algorithm's Plutchik mapping verified?
- [ ] Have you run `uv run pytest` on the new module?
- [ ] Is latency within the <200ms window?
- [ ] Are PII discovery path filters active?
- [ ] Does the score normalization mapping match the global 0-1 scale?
